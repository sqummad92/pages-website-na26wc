# pages-website-na26wc

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态文件的形式存储，不依赖特定域名或后端服务，适合用于展示、测试或存档用途。

仓库结构清晰，方便直接通过 GitHub Pages 或其他静态托管服务进行浏览。

## 目录说明

```
pages-website-na26wc/
├── index.html          # 项目入口页面，可提供页面列表或导航
├── pages/              # 存放各个独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 公共资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- **index.html**：项目首页，可作页面入口或索引。
- **pages/**：各个 HTML 页面文件，彼此独立，可单独访问。
- **assets/**：存放页面所需的公共静态资源，按类型分目录整理。

## 页面归档说明

- 每个 HTML 页面均为独立文件，可自由修改、替换或新增。
- 页面之间无硬性依赖，便于单独维护和部署。
- 若需添加新页面，请将文件放入 `pages/` 目录，并在 `index.html` 中更新链接（如有需要）。

## 维护说明

- 欢迎提交 Pull Request 添加或修改页面。
- 请保持文件命名简洁，避免特殊字符。
- 若页面引用外部资源，建议使用相对路径或 CDN 链接，确保可移植性。
- 本仓库不涉及商业用途，仅作页面归档与展示。
